通过本论文的第三章中的需求分析后，对用户想要什么样的问卷系统已经有了清晰的了解，且抽离出组成系统的各个功能模块。本论文采取经典前后端分离架构，基于 B/S 模式设计，所以问卷系统的系统架构设计主要分为三个层次：前端 UI 层、后端处理层和数据存储层。前端使用 Vue3 构建单页面应用，后端使用 Node.js 和 Exoress 框架提供 RESTful API 服务，数据存储采用 MYSQL。前端发送异步请求获取数据，后端处理业务并操作数据库数据，数据库负责数据的存储。以下将详细说明这三部分的设计。
1、前端 UI
前端 UI 层作为用户直接交互的界面，主要负责页面展示、交互逻辑处理以及与后端的数据通信。本系统的前端采用 Vue 3 框架，结合 Element Plus 组件库构建统一的页面风格，保障系统界面友好、清晰、操作流畅。
前端 UI 层的主要功能包括：问卷浏览、问卷填写、问卷创建与编辑、收藏与评论、个人中心管理、积分与成就展示、AI 报告查看等模块。用户可以根据自身需求在界面上进行对应操作，系统能通过动态渲染实时更新界面内容，提升交互体验。。
2、后端业务处理层
后端业务处理层主要负责接收前端传输的数据请求，执行对应业务逻辑，并将结果返回给前端。后端基于 Node.js + Express 构建，并按照业务模块划分控制器。在业务处理过程中，后端需要完成用户身份验证、问卷创建管理、答卷处理、评论管理、积分与成就计算、管理员审核等功能。例如在用户登录时，后端接收前端的账号和密码，通过数据库进行比对后返回登录成功或失败的结果；在问卷发布时，后端对问卷内容进行校验后将结果写入数据层。此外，后端还负责数据安全校验、权限控制、错误处理等工作，以确保系统数据的安全与可控性。
3、数据存储层
数据存储层主要负责存储用户信息、问卷数据、题目内容、答卷记录、评价信息、积分记录、徽章成就等数据。
